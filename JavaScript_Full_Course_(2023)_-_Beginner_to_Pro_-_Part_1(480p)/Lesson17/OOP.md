# Object Oriented Programming (OOP)

## What is OOP?

OOP is a programming paradigm that uses "objects" to design applications. Objects can contain data in the form of fields (often known as attributes or properties), and code, in the form of procedures (often known as methods or functions).

## Why OOP?

OOP helps us to organize our code in a way that is easy to understand, maintain, and reuse. It also helps us to model real-world objects in our code.

## Key Concepts of OOP

1. **Objects**: Objects are instances of classes. They have properties and methods.
2. **Classes**: Classes are blueprints for creating objects. They define the properties and methods that objects will have.
3. **Inheritance**: Inheritance is a mechanism that allows us to create a new class (subclass) that inherits properties and methods from an existing class (superclass).
4. **Polymorphism**: Polymorphism is a mechanism that allows us to create a new class (subclass) that inherits properties and methods from an existing class (superclass).
5. **Encapsulation**: Encapsulation is a mechanism that allows us to create a new class (subclass) that inherits properties and methods from an existing class (superclass).
6. **Abstraction**: Abstraction is a mechanism that allows us to create a new class (subclass) that inherits properties and methods from an existing class (superclass).

## OOP in JavaScript

JavaScript is a prototype-based language, which means that it uses objects to implement OOP concepts.
